## Lessons Learned

### What Went Well ✨

1. **"코드 path 1개" 원칙의 가치**
   - 순차 로직을 group_order=특수케이스(크기=1)로 자연 수용
   - v1.0/v1.1 회귀 0건, 기존 smoke 40/40 유지
   - `fn_advance_document` helper가 두 방식을 완벽히 커버

2. **`FormRenderer` 재사용의 강력함**
   - 화면 = PDF 픽셀 100% 일치 (렌더 로직 1개)
   - 폰트/레이아웃/색상 동기화 부담 0
   - Chromium headless mode + CSS `@media print` 만으로 충분

3. **JCS RFC 8785 자체 구현**
   - 외부 의존성 0, npm package 1개 추가 불필요
   - ~60줄 코드, 단위 테스트 철저
   - 클라이언트/서버 동일 로직 공유 → 감사 신뢰도 상승

4. **puppeteer 싱글톤 풀의 우아함**
   - boot-time warm-up 1회 → 콜드스타트 3s → 2회차 1.5s
   - 메모리 peak ≤ 400MB (Node 전체 RSS)
   - 프로세스 pool 구현 불필요 (싱글톤으로 충분)

5. **audit_logs payload mirror 패턴**
   - 서명/해시는 audit 시점에 snapshot → 이후 profile 변경 영향 0
   - 감사 재현성 최고 수준
   - "증빙 3종"이 하나의 원자적 기록으로 완성

### Areas for Improvement 🎯

1. **Design Doc Drift Prevention (프로세스화 필요)**
   - 현재: `pg_get_functiondef` 캡처는 수동 (M17에서 미수행)
   - v1.3: Migration apply 직후 자동 스크립트로 ERRATA 생성 추천
   - 예: `supabase_after_apply.sh` hook에서 모든 function signature capture

2. **PDF Retry/Fallback (Low Priority)**
   - 503 에러 시 단순 1회 재시도만 구현
   - 실무: puppeteer timeout / memory spike 시 graceful degrade 고려

3. **Group Model 확장성 검토 (v1.3 계획)**
   - 현재 v1.2: AND-only (모든 그룹 완료)
   - v1.3: OR / K-of-N 필요 시 D1-B(groups table) 마이그레이션 검토
   - 현 column 모델로는 group metadata (completion_condition) 부착 불가

4. **DocumentSteps Comment Filter 정책 명확화**
   - 설계: 전결 코멘트 숨김
   - 구현: 모든 코멘트 표시
   - v1.3: 사용자 피드백 후 정책 확정 필요

5. **Form FOR UPDATE 보험 (Medium Priority)**
   - v1.2 G7 accepted risk (SMB context)
   - v1.3: 트래픽 증가 후 동시 form publish race 감시 권장

### To Apply Next Time 📋

1. **ERRATA 프로세스 자동화**
   - Migration apply 직후 모든 RPC signature capture → ERRATA.md 자동 생성
   - Design doc의 "RPC Signature" 섹션과 ERRATA 동기화 스크립트
   
2. **Helper 함수 추출 원칙 문서화**
   - "한 가지 로직이 3곳 이상 반복되면 HELPER로 추출" 규칙 명시
   - v1.1 G4 이월 사태 재발 방지 (지금은 v1.2 plan에 의무 포함)

3. **Smoke Test 자동 생성 가이드**
   - Design doc §11 "Test Plan"의 각 요구사항 → SQL smoke 1건 매핑
   - S40~S47 같은 명명 규칙 준수 → 자동 커버리지 리포트
   
4. **puppeteer Boot-up 부하 테스트**
   - 향후 large PDF (100+ pages) 렌더 시 memory/timeout 재측정 필수
   - 현재 warm-up은 빈 페이지. 실제 문서 크기 다양성 고려

5. **RLS Policy 4개 체크리스트** (Signature Storage 패턴)

   - "본인만 upsert" + "tenant 멤버 read" 같은 표준 패턴 → 재사용 가능 스니펫

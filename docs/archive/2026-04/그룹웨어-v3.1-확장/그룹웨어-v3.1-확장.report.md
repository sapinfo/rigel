## 4. 기술 하이라이트

### 4.1 RLS 규칙 준수 확인

- `(SELECT auth.uid())` InitPlan 래핑: 모든 신규 RLS 정책 적용
- `FOR ALL` 미사용: action별 SELECT/INSERT/UPDATE/DELETE 분리
- SECURITY DEFINER 헬퍼: `can_access_board`, `can_view_event`
- 양방향 EXISTS 방지: board_posts → boards는 SECURITY DEFINER 경유

### 4.2 기존 인프라 재활용

| 기존 자산 | 재활용 |
|---|---|
| RichEditor (Tiptap) | 게시글/공지 본문 |
| AdminNav | 메뉴 확장만 |
| OrgTreePicker | 부서 게시판 생성 |
| uploadAttachment 패턴 | uploadBoardFile 구조 동일 |
| approval_forms | 초과근무 양식 seed |

### 4.3 주요 설계 결정

| 결정 | 근거 |
|---|---|
| 사이드바 전환 | 메뉴 수 12+ → 수평 nav 한계 |
| attendance_settings 선행 생성 | fn_clock_in이 참조 → DDL 순서 |
| EXCLUDE constraint (회의실) | DB 레벨 시간 겹침 방지 (race-free) |
| 팝업 공지 최대 3개 | 클라이언트 제한 (UX 피로 방지) |

## 2. Data Flow

### 2.1 알림 생성 흐름

```
사용자 액션 (approve/reject/submit/comment/withdraw)
  → approval RPC (fn_approve_step / fn_submit_draft / …)
  → INSERT INTO approval_audit_logs
  → TRIGGER: fn_create_notification()
      ├─ 수신자 결정 (action 유형별 로직)
      ├─ INSERT INTO notifications (수신자별)
      └─ pg_net.http_post → Edge Function (비동기)
```

### 2.2 Realtime 구독 흐름

```
+layout.svelte onMount
  → supabase.channel('notif:' + userId)
      .on('postgres_changes', { event: 'INSERT', table: 'notifications',
           filter: 'user_id=eq.' + userId })
      .subscribe()
  → 새 row 수신
      → notifications 배열 prepend
      → unreadCount++
      → bell badge 업데이트
```

### 2.3 이메일 발송 흐름

```
fn_create_notification() (DB trigger)
  → pg_net.http_post(
       url  = EDGE_FUNCTION_URL || '/send-notification-email',
       body = {
         notification_id, tenant_id, user_id,
         type, title, body, document_id
       }
     )  -- fire-and-forget (결과 무시)

Edge Function (Deno)
  → Supabase service_role 클라이언트로 recipient 이메일 조회
  → 알림 유형별 한국어 템플릿 렌더링
  → SMTP_HOST 환경변수 존재 시 이메일 발송
  → 미설정 시 console.log로 fallback
```

### 2.4 읽음 처리 흐름

```
사용자 클릭 (NotificationDropdown 항목 또는 "모두 읽음")
  → supabase.rpc('mark_notification_read', { p_notification_id })
       or supabase.rpc('mark_all_notifications_read', { p_tenant_id })
  → 해당 문서로 navigate (단건 클릭 시)
  → $state 로컬 업데이트 (RPC 결과 기다리지 않고 즉시 반영)
```

## 5. Trigger Logic — fn_create_notification

### 5.1 전체 함수 스펙

```sql
-- 0038_notifications.sql 의 일부
CREATE OR REPLACE FUNCTION public.fn_create_notification()
RETURNS trigger
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
DECLARE
  v_doc       public.approval_documents;
  v_action    text := NEW.action::text;
  v_doc_id    uuid := NEW.document_id;
  v_tenant_id uuid := NEW.tenant_id;
  v_actor_id  uuid := NEW.actor_id;
  v_current_group int;
  v_recipient uuid;
  v_notif_type public.notification_type;
  v_title     text;
  v_body      text;
  v_doc_title text;
  r           record;
BEGIN
  -- draft_saved, delegated, auto_delegated, approved_by_proxy(내부 처리)는 알림 불필요
  -- approved_by_proxy는 approve와 동일한 경로로 처리 (아래 approve 분기에서 통합)
  IF v_action IN ('draft_saved', 'delegated', 'auto_delegated', 'submitted_post_facto') THEN
    RETURN NEW;
  END IF;

  -- 문서 조회
  SELECT * INTO v_doc
    FROM public.approval_documents
   WHERE id = v_doc_id;

  IF v_doc.id IS NULL THEN
    RETURN NEW;  -- 문서 없으면 조용히 반환
  END IF;

  -- 문서 제목: form_schema_snapshot에서 추출 시도, fallback은 doc_number
  v_doc_title := coalesce(
    v_doc.form_schema_snapshot->>'title',
    v_doc.doc_number
  );

  v_current_group := v_doc.current_step_index;

  -- ── 액션별 수신자 + 알림 유형 결정 ──────────────────────────────

  IF v_action = 'submit' THEN
    -- 수신자: 현재 그룹(current_step_index)의 pending approval step 결재자들
    v_notif_type := 'approval_requested';
    v_title      := format('[결재 요청] %s', v_doc_title);
    v_body       := format('%s님이 결재를 요청했습니다.', v_actor_id::text);

    FOR r IN
      SELECT approver_user_id
        FROM public.approval_steps
       WHERE document_id = v_doc_id
         AND group_order  = v_current_group
         AND step_type    = 'approval'
         AND status       = 'pending'
    LOOP
      INSERT INTO public.notifications
        (tenant_id, user_id, document_id, type, title, body)
      VALUES
        (v_tenant_id, r.approver_user_id, v_doc_id, v_notif_type, v_title, v_body);
    END LOOP;

  ELSIF v_action IN ('approve', 'approved_by_proxy') THEN
    -- 문서가 completed → 기안자에게 "결재 완료" 알림
    IF v_doc.status = 'completed' THEN
      v_notif_type := 'approved';
      v_title      := format('[결재 완료] %s', v_doc_title);
      v_body       := '문서가 최종 승인되었습니다.';
      INSERT INTO public.notifications
        (tenant_id, user_id, document_id, type, title, body)
      VALUES
        (v_tenant_id, v_doc.drafter_id, v_doc_id, v_notif_type, v_title, v_body);
    ELSE
      -- 다음 그룹 결재자에게 "결재 요청" 알림
      v_notif_type := 'approval_requested';
      v_title      := format('[결재 요청] %s', v_doc_title);
      v_body       := '이전 결재자가 승인하였습니다. 결재를 진행해 주세요.';

      FOR r IN
        SELECT approver_user_id
          FROM public.approval_steps
         WHERE document_id = v_doc_id
           AND group_order  = v_current_group
           AND step_type    = 'approval'
           AND status       = 'pending'
      LOOP
        INSERT INTO public.notifications
          (tenant_id, user_id, document_id, type, title, body)
        VALUES
          (v_tenant_id, r.approver_user_id, v_doc_id, v_notif_type, v_title, v_body);
      END LOOP;
    END IF;

  ELSIF v_action = 'reject' THEN
    -- 수신자: 기안자
    v_notif_type := 'rejected';
    v_title      := format('[반려] %s', v_doc_title);
    v_body       := '문서가 반려되었습니다.';
    INSERT INTO public.notifications
      (tenant_id, user_id, document_id, type, title, body)
    VALUES
      (v_tenant_id, v_doc.drafter_id, v_doc_id, v_notif_type, v_title, v_body);

  ELSIF v_action = 'comment' THEN
    -- 수신자: 기안자 + 모든 결재자 (액터 본인 제외)
    v_notif_type := 'commented';
    v_title      := format('[코멘트] %s', v_doc_title);
    v_body       := '새 코멘트가 등록되었습니다.';

    -- 기안자 (액터가 아닌 경우)
    IF v_doc.drafter_id <> v_actor_id THEN
      INSERT INTO public.notifications
        (tenant_id, user_id, document_id, type, title, body)
      VALUES
        (v_tenant_id, v_doc.drafter_id, v_doc_id, v_notif_type, v_title, v_body);
    END IF;

    -- 모든 approval step 담당자 (중복 + 액터 제외)
    FOR r IN
      SELECT DISTINCT approver_user_id
        FROM public.approval_steps
       WHERE document_id = v_doc_id
         AND step_type   = 'approval'
         AND approver_user_id <> v_actor_id
         AND approver_user_id <> v_doc.drafter_id  -- 이미 위에서 추가
    LOOP
      INSERT INTO public.notifications
        (tenant_id, user_id, document_id, type, title, body)
      VALUES
        (v_tenant_id, r.approver_user_id, v_doc_id, v_notif_type, v_title, v_body);
    END LOOP;

  ELSIF v_action = 'withdraw' THEN
    -- 수신자: 현재 그룹 pending 결재자들
    v_notif_type := 'withdrawn';
    v_title      := format('[회수] %s', v_doc_title);
    v_body       := '기안자가 문서를 회수하였습니다.';

    FOR r IN
      SELECT approver_user_id
        FROM public.approval_steps
       WHERE document_id = v_doc_id
         AND group_order  = v_current_group
         AND step_type    = 'approval'
         AND status       = 'pending'
    LOOP
      INSERT INTO public.notifications
        (tenant_id, user_id, document_id, type, title, body)
      VALUES
        (v_tenant_id, r.approver_user_id, v_doc_id, v_notif_type, v_title, v_body);
    END LOOP;
  END IF;

  -- ── pg_net: Edge Function 이메일 호출 (fire-and-forget) ─────
  -- 0039 migration에서 pg_net extension 설치 후 활성화.
  -- 개발 환경에서 pg_net 미설치 시 skip.
  PERFORM public.fn_notify_email_async(v_doc_id, v_tenant_id, v_action);

  RETURN NEW;
EXCEPTION WHEN OTHERS THEN
  -- 알림 실패가 결재 흐름을 막으면 안 됨 → 예외 삼킴
  RAISE WARNING 'fn_create_notification error: % %', SQLERRM, SQLSTATE;
  RETURN NEW;
END;
$$;

GRANT EXECUTE ON FUNCTION public.fn_create_notification() TO authenticated;

CREATE TRIGGER trg_create_notification
  AFTER INSERT ON public.approval_audit_logs
  FOR EACH ROW
  EXECUTE FUNCTION public.fn_create_notification();
```

```sql
CREATE EXTENSION IF NOT EXISTS pg_net;

-- Edge Function URL 저장 (환경별 설정)
CREATE TABLE IF NOT EXISTS public.app_settings (
  key   text PRIMARY KEY,
  value text NOT NULL
);

INSERT INTO public.app_settings (key, value)
  VALUES ('edge_function_base_url', 'http://host.docker.internal:54321/functions/v1')
  ON CONFLICT (key) DO NOTHING;

-- fn_notify_email_async 실제 구현
CREATE OR REPLACE FUNCTION public.fn_notify_email_async(
  p_document_id uuid,
  p_tenant_id   uuid,
  p_action      text
)
RETURNS void
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
DECLARE
  v_base_url text;
  v_url      text;
  v_payload  jsonb;
BEGIN
  SELECT value INTO v_base_url
    FROM public.app_settings
   WHERE key = 'edge_function_base_url';

  IF v_base_url IS NULL THEN
    RETURN;
  END IF;

  v_url := v_base_url || '/send-notification-email';

  -- 최근 삽입된 notifications 배치 전송 (현재 document + tenant)
  v_payload := jsonb_build_object(
    'document_id', p_document_id,
    'tenant_id',   p_tenant_id,
    'action',      p_action
  );

  PERFORM net.http_post(
    url     := v_url,
    body    := v_payload::text,
    headers := '{"Content-Type": "application/json"}'::jsonb
  );
EXCEPTION WHEN OTHERS THEN
  RAISE WARNING 'fn_notify_email_async error: %', SQLERRM;
END;
$$;
```

## 6. RPC Definitions

### 6.1 mark_notification_read

```sql
CREATE OR REPLACE FUNCTION public.mark_notification_read(
  p_notification_id uuid
)
RETURNS void
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
BEGIN
  UPDATE public.notifications
     SET read = true
   WHERE id      = p_notification_id
```

```sql
  p_tenant_id uuid
)
RETURNS int  -- 업데이트된 행 수 반환

SECURITY DEFINER
SET search_path = public
AS $$
DECLARE
  v_count int;
BEGIN
  UPDATE public.notifications
     SET read = true
   WHERE user_id   = (SELECT auth.uid())
     AND tenant_id = p_tenant_id
     AND read      = false;

  GET DIAGNOSTICS v_count = ROW_COUNT;
  RETURN v_count;
END;
```

```sql
CREATE OR REPLACE FUNCTION public.get_notifications(
  p_tenant_id uuid,
  p_limit     int DEFAULT 20,
  p_offset    int DEFAULT 0
)
RETURNS TABLE (
  id          uuid,
  document_id uuid,
  type        public.notification_type,
  title       text,
  body        text,
  read        boolean,
  created_at  timestamptz
)
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
BEGIN
  RETURN QUERY
    SELECT n.id, n.document_id, n.type, n.title, n.body, n.read, n.created_at
      FROM public.notifications n
     WHERE n.user_id   = (SELECT auth.uid())
       AND n.tenant_id = p_tenant_id
     ORDER BY n.created_at DESC
     LIMIT  p_limit
     OFFSET p_offset;
END;
$$;
```

```typescript
    .from('tenant_members')
    .select('role, tenant:tenants!inner(id, slug, name)')
    .eq('user_id', locals.user.id)
    .eq('tenant.slug', params.tenantSlug)
    .maybeSingle();

  if (err || !data) {
    error(404, 'Tenant not found or you are not a member');
  }

  const tenant = data.tenant as unknown as { id: string; slug: string; name: string };
  const currentTenant: CurrentTenant = {
    id: tenant.id,
    slug: tenant.slug,
    name: tenant.name,
    role: data.role as TenantRole
  };

  locals.currentTenant = currentTenant;

  // Published forms list for the "New draft" dropdown
  const { data: formRows } = await locals.supabase
    .from('approval_forms')
    .select('code, name')
    .eq('tenant_id', currentTenant.id)
    .eq('is_published', true)
    .order('name', { ascending: true });

  const forms = (formRows ?? []).map((f) => ({
    code: f.code as string,
    name: f.name as string
  }));

  // v1.3: SSR initial unread count (인앱 bell 초기값)
  const { count } = await locals.supabase
    .from('notifications')
    .select('id', { count: 'exact', head: true })
    .eq('user_id', locals.user.id)
    .eq('tenant_id', currentTenant.id)
    .eq('read', false);

  const unreadCount = count ?? 0;

  return { currentTenant, forms, unreadCount };
```

```typescript
  const tenantSlug = tenantRow?.slug ?? '';

  let sent = 0;
  for (const notif of notifs) {
    const email = emailMap.get(notif.user_id);
    if (!email) continue;

    const subject = SUBJECT_MAP[notif.type] ?? '[Rigel] 알림';
    const docUrl  = `${appBaseUrl}/t/${tenantSlug}/approval/documents/${notif.document_id}`;
    const html    = renderHtml(notif.title, notif.body, docUrl);

    try {
      await sendEmail(email, subject, html);
      sent++;
    } catch (e) {
      console.error(`[send-notification-email] Failed to send to ${email}:`, e);
    }
  }

  return new Response(JSON.stringify({ ok: true, sent }), {
    status: 200,
    headers: { 'Content-Type': 'application/json' }
  });
});
```

### 10.3 src/service-worker.ts

SvelteKit 빌트인 service worker 지원 활용. v1.3 scope: 홈스크린 설치만 (오프라인 캐시는 v1.4).

```typescript
// src/service-worker.ts
/// <reference types="@sveltejs/kit" />
/// <reference no-default-lib="true"/>
/// <reference lib="esnext" />
/// <reference lib="webworker" />

import { build, files, version } from '$service-worker';

const sw = self as unknown as ServiceWorkerGlobalScope;

const CACHE_NAME = `rigel-v${version}`;

// install: network-first 전략이므로 static assets만 precache
sw.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME).then((cache) => {
      // SvelteKit 빌드 산출물 + 정적 파일 캐싱
      return cache.addAll([...build, ...files]);
    })
  );
  sw.skipWaiting();
});

sw.addEventListener('activate', (event) => {
  // 이전 버전 캐시 정리
  event.waitUntil(
    caches.keys().then((keys) =>
      Promise.all(
        keys.filter((key) => key !== CACHE_NAME).map((key) => caches.delete(key))
      )
    )
  );
  sw.clients.claim();
});

// fetch: network-first (API, SSR 우선 네트워크)
sw.addEventListener('fetch', (event) => {
  if (event.request.method !== 'GET') return;

  // Supabase API 요청은 캐시하지 않음
  const url = new URL(event.request.url);
  if (url.pathname.startsWith('/rest/') || url.pathname.startsWith('/auth/')) return;

  event.respondWith(
    fetch(event.request)
      .then((response) => {
        const clone = response.clone();
        caches.open(CACHE_NAME).then((cache) => cache.put(event.request, clone));
        return response;
      })
      .catch(() => caches.match(event.request).then((cached) => cached ?? Response.error()))
  );
});
```

## 12. Error Handling

| 상황 | 처리 방식 |
|---|---|
| fn_create_notification 예외 | EXCEPTION WHEN OTHERS → RAISE WARNING → RETURN NEW (결재 흐름 차단 없음) |
| pg_net 호출 실패 | fn_notify_email_async 내 EXCEPTION → RAISE WARNING → 조용히 반환 |
| Edge Function cold start / 타임아웃 | fire-and-forget (DB는 결과 기다리지 않음). 이메일 누락 가능하나 인앱 알림은 정상 |
| Edge Function 이메일 발송 실패 | try/catch → console.error → 다른 수신자 계속 시도 |
| Realtime 연결 끊김 | Supabase JS SDK 자동 재연결. unreadCount는 SSR initial값 유지 |
| mark_notification_read RPC 실패 | $state는 낙관적 업데이트 → 실패해도 UI는 읽음 표시 유지 (다음 SSR reload 시 보정) |
| 알림 개수 불일치 (SSR vs realtime) | layout reload 시 unreadCount 재확인. `invalidate('app:notifications')` 패턴 사용 가능 |

## 13. Security Considerations

| 항목 | 설계 결정 |
|---|---|
| notifications RLS | SELECT: `user_id = (SELECT auth.uid())`. INSERT는 정책 없음 → SECURITY DEFINER 함수만 삽입 가능 |
| fn_create_notification | SECURITY DEFINER + search_path=public. RLS 우회하여 타인 알림 삽입 가능 (의도적) |
| mark_notification_read | SECURITY DEFINER이나 내부에서 `user_id = (SELECT auth.uid())` 조건 — 타인 알림 읽음 처리 불가 |
| get_notifications | SECURITY DEFINER + user_id 필터 — 타인 알림 조회 불가 |
| Edge Function | service_role 키 사용 → Supabase secrets에 저장. 코드에 hardcode 금지 |
| pg_net 호출 | payload에 notification 내용 포함하지 않음 (document_id + tenant_id + action만) → Edge Function이 DB에서 직접 조회 |
| Realtime filter | `filter: user_id=eq.{userId}` — 서버 side RLS + 클라이언트 filter 이중 적용 |
| auth.uid() 래핑 | 모든 RPC에서 `(SELECT auth.uid())` InitPlan 방식 사용 (CLAUDE.md 규칙 준수) |

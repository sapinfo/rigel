## 3. Route & Action Design

### 3.1 `/admin/org/+page.server.ts` (M3, M4 — 전면 개편)

**load**:

```typescript
// 1. 부서 전체 (트리 구성용)
const { data: depts } = await supabase
  .from('departments')
  .select('id, name, path, parent_id, sort_order')
  .eq('tenant_id', tenantId)
  .order('sort_order').order('name');

// 2. 직급 전체
const { data: jobTitles } = await supabase
  .from('job_titles')
  .select('id, name, level')
  .eq('tenant_id', tenantId)
  .order('level', { ascending: true });

// 3. 멤버 (부서별 count + 직급 할당용)
const { data: members } = await supabase
  .from('tenant_members')
  .select('user_id, department_id, job_title_id')
  .eq('tenant_id', tenantId);

// 4. 프로필 (2-query join)
const profileMap = await loadProfileMap(supabase, userIds);

return { departments, jobTitles, members, profileMap };
```

**actions**:

| Action | 파라미터 | 동작 |
|---|---|---|
| `createDept` | name, parentId? | INSERT departments (path 자동 생성) |
| `updateDept` | id, name | UPDATE name + path |
| `reparent` | id, newParentId | UPDATE parent_id + path 재계산. 순환 참조 검사 필수 |
| `removeDept` | id | 하위 부서 존재 시 FAIL. 없으면 DELETE |
| `createJobTitle` | name, level | INSERT job_titles |
| `updateJobTitle` | id, name?, level? | UPDATE job_titles |
| `removeJobTitle` | id | DELETE (FK로 할당된 멤버 있으면 경고) |
| `assignJobTitle` | userId, jobTitleId | UPDATE tenant_members SET job_title_id |
| `assignDept` | userId, deptId | UPDATE tenant_members SET department_id |

**순환 참조 검사** (`reparent` action):
```typescript
// 서버에서 ancestor 체크: newParentId의 조상 중 id가 있으면 거부
async function hasAncestor(supabase, deptId: string, ancestorId: string): Promise<boolean> {
  let current = deptId;
  const visited = new Set<string>();
  while (current) {
    if (current === ancestorId) return true;
    if (visited.has(current)) return true; // 기존 cycle 감지
    visited.add(current);
    const { data } = await supabase
      .from('departments').select('parent_id').eq('id', current).single();
    current = data?.parent_id;
  }
  return false;
}
```

```typescript
simulate: async ({ request, locals, params }) => {
  // 입력: formId, drafterId (선택), sampleContent (JSON)
  // 1. 기안자의 tenant_members 조회 (dept, job_title)
  // 2. fn_resolve_approval_line RPC 호출 (service_role 불필요 — 기존 RLS OK)
  //    단, auth.uid()가 호출자이므로 drafterId를 지정하려면
  //    → 대안: 서버에서 직접 SQL 로직 재현 (RPC는 auth.uid() 기반)
  //    → 결정: simulate는 "현재 로그인 관리자 기준"으로 호출.
  //      다른 기안자 시뮬레이션은 v2.2 (service_role RPC 필요)
  // 3. 결과 userId[] → profile join → ResolvedApproverInfo[] 반환

  const { data: result } = await locals.supabase
    .rpc('fn_resolve_approval_line', {
      p_tenant_id: tenantId,
      p_form_id: formId,
      p_content: sampleContent
    });

  // profile 조회 + join
  const approvers = await resolveApproverProfiles(locals.supabase, result);

  // 매칭된 규칙 이름 조회
  const ruleName = await findMatchedRuleName(locals.supabase, tenantId, formId);

  return { simulateResult: { approvers, ruleName, error: null } };
}
```

> **제약**: fn_resolve_approval_line은 `auth.uid()` 기반. 관리자가 다른 기안자로 시뮬레이션하려면 별도 RPC 필요 → v2.2 범위.

### 3.3 `/approval/drafts/new/[formCode]/+page.server.ts` (M2)

**신규 action `resolvePreview`**:

```typescript
resolvePreview: async ({ request, locals, params }) => {
  if (!locals.user) return fail(401);
  const currentTenant = await resolveTenant(locals.supabase, locals.user.id, params.tenantSlug);
  if (!currentTenant) return fail(404);

  const fd = await request.formData();
  const formId = fd.get('formId')?.toString();
  const content = parseJson(fd.get('content'), {});

  if (!formId) return fail(400, { errors: formError('양식 ID 필수') });

  // 1. fn_resolve_approval_line 호출
  const { data: resolved, error: rpcErr } = await locals.supabase
    .rpc('fn_resolve_approval_line', {
      p_tenant_id: currentTenant.id,
      p_form_id: formId,
      p_content: content
    });

  if (rpcErr) {
    return fail(400, { previewError: rpcErr.message });
  }

  const lineItems = (resolved ?? []) as { userId: string; stepType: string }[];

  if (lineItems.length === 0) {
    return { preview: { approvers: [], ruleName: null, error: '매칭되는 결재선 규칙이 없습니다. 관리자에게 문의하세요.' } };
  }

  // 2. userId[] → profile + dept + jobTitle 조인
  const userIds = lineItems.map(i => i.userId);
  const { data: profiles } = await locals.supabase
    .from('profiles').select('id, display_name, email').in('id', userIds);
  const { data: tmRows } = await locals.supabase
    .from('tenant_members')
    .select('user_id, department_id, job_title_id')
    .eq('tenant_id', currentTenant.id)
    .in('user_id', userIds);
  const { data: depts } = await locals.supabase
    .from('departments').select('id, name').eq('tenant_id', currentTenant.id);
  const { data: jts } = await locals.supabase
    .from('job_titles').select('id, name').eq('tenant_id', currentTenant.id);

  const profileMap = new Map((profiles ?? []).map(p => [p.id, p]));
  const tmMap = new Map((tmRows ?? []).map(r => [r.user_id, r]));
  const deptMap = new Map((depts ?? []).map(d => [d.id, d.name]));
  const jtMap = new Map((jts ?? []).map(j => [j.id, j.name]));

  const approvers: ResolvedApproverInfo[] = lineItems.map(item => {
    const p = profileMap.get(item.userId);
    const tm = tmMap.get(item.userId);
    return {
      userId: item.userId,
      displayName: p?.display_name ?? '(알 수 없음)',
      email: p?.email ?? '',
      departmentName: tm?.department_id ? (deptMap.get(tm.department_id) ?? null) : null,
      jobTitleName: tm?.job_title_id ? (jtMap.get(tm.job_title_id) ?? null) : null,
      stepType: item.stepType as StepType
    };
  });

  // 3. 매칭 규칙 이름 (가장 높은 우선순위)
  const { data: matchedRule } = await locals.supabase
    .from('approval_rules')
    .select('name')
    .eq('tenant_id', currentTenant.id)
    .eq('is_active', true)
    .or(`form_id.eq.${formId},form_id.is.null`)
    .order('form_id', { ascending: false, nullsFirst: false })
    .order('priority', { ascending: true })
    .limit(1)
    .maybeSingle();

  return {
    preview: {
      approvers,
      ruleName: matchedRule?.name ?? null,
      error: null
    }
  };
}
```

**트리 빌드 로직** (클라이언트 `+page.svelte`):

```typescript
function buildTree(depts: Dept[]): DeptTreeNode[] {
  const map = new Map<string, DeptTreeNode>();
  for (const d of depts) {
    map.set(d.id, { ...d, parentId: d.parent_id, memberCount: 0, children: [] });
  }
  const roots: DeptTreeNode[] = [];
  for (const node of map.values()) {
    if (node.parentId && map.has(node.parentId)) {
      map.get(node.parentId)!.children.push(node);
    } else {
      roots.push(node);
    }
  }
  // memberCount 계산
  for (const m of members) {
    if (m.departmentId && map.has(m.departmentId)) {
      map.get(m.departmentId)!.memberCount++;
    }
  }
  return roots;
}
```

### 4.6 M6: ConditionalsEditor.svelte (FormBuilder 확장)

```svelte
<script lang="ts">
  import type { ConditionalRule, FormField } from '$lib/types/approval';

  type Props = {
    fields: FormField[];
    conditionals: ConditionalRule[];
    onChange: (next: ConditionalRule[]) => void;
  };
  let { fields, conditionals, onChange }: Props = $props();

  function addRule() {
    onChange([...conditionals, {
      if: { fieldId: '', operator: 'eq', value: '' },
      show: [],
      hide: []
    }]);
  }

  function removeRule(idx: number) {
    onChange(conditionals.filter((_, i) => i !== idx));
```
